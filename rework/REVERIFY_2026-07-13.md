### 2.1 ACTION REQUIRED — the deployed backend is stale (highest priority)
The live backend gunicorn on **:8012 is the same process (pid 497892) from before
the rework** — it is serving **pre-rework code**. Proof: `GET :8012/…/instructor/
sc-panel/` → **404**. And the served frontend on **:8014 proxies `/api` to :8012**,
so **the app a user actually opens right now would 404 on every new feature**
(instructor panel, FX, compliance) even though the frontend build is fresh and
contains them.

**This is a deployment action, not a code gap.** Restart the globalstrat+ backend
(reload gunicorn on :8012) so the running service matches the merged code, then
re-run the §3 browser pass against the live stack. I did **not** restart it myself
(it's a shared service; restarting unprompted isn't an auditor's call). I verified
the new code by running a fresh `runserver` on :8016 instead — everything there is
correct.

- **Instructor panel doesn't list *pending* injections.** After injecting, the API
  returns `"queued — fires when round N is advanced"`, but the panel's

  instructor gets no in-panel confirmation that a queued injection is staged.
  Consider surfacing pending (`resolution_data.pending`) injections on the panel.
- **`JWT_SECRET_KEY = SECRET_KEY`** — harmless in prod now (fail-closed), but a
  dedicated JWT key would be cleaner defence-in-depth.

### 2.3 Honestly-deferred, still not built (by design — fine for v1, name them so they aren't forgotten)
